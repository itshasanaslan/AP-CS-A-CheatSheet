Classical vs Enhanced For Loops

Feature Classical For Loop Enhanced For Loop
Syntax for (int i = 0; i < array.length; i++) for (int element : array)
Index Access Direct access to index (i) No direct index access
Element Modification Can modify elements Cannot modify elements
Reverse Traversal Possible Not possible
Use Cases When index needed or reverse traversal required When only reading elements sequentially 

// Example: Classical vs Enhanced For Loops
public class LoopComparison {
  public static void main(String[] args) {
    int[] numbers = {1, 2, 3, 4, 5};
    
    // Classical for loop
    System.out.println("Classical for loop:");
    for (int i = 0; i < numbers.length; i++) {
      System.out.println("Index " + i + ": " + numbers[i]);
      numbers[i] *= 2; // Can modify elements
    }
    
    // Enhanced for loop
    System.out.println("\nEnhanced for loop:");
    for (int num : numbers) {
      System.out.println(num);
      // num *= 2; // This won't modify the array
    }
  }
}

Example Code


// Example: Working with Arrays in Java
public class ArrayExamples {
  public static void main(String[] args) {
    // Array declaration and initialization
    int[] numbers = {1, 2, 3, 4, 5};
    String[] names = new String[3];
    names[0] = "Alice";
    names[1] = "Bob";
    names[2] = "Charlie";
    
    // Array traversal
    System.out.println("Numbers array:");
    for (int i = 0; i < numbers.length; i++) {
      System.out.println("Index " + i + ": " + numbers[i]);
    }
    
    // Enhanced for loop
    System.out.println("\nNames array:");
    for (String name : names) {
      System.out.println(name);
    }
    
    // Array as parameter
    int sum = calculateSum(numbers);
    System.out.println("\nSum of numbers: " + sum);
    
    // Array algorithms
    int max = findMax(numbers);
    System.out.println("Maximum value: " + max);
    
    // Array bounds
    try {
      System.out.println("Accessing index 5: " + numbers[5]);
    } catch (ArrayIndexOutOfBoundsException e) {
      System.out.println("Error: Array index out of bounds");
    }
  }
  
  // Method with array parameter
  public static int calculateSum(int[] arr) {
    int sum = 0;
    for (int num : arr) {
      sum += num;
    }
    return sum;
  }
  
  // Finding maximum value
  public static int findMax(int[] arr) {
    int max = arr[0];
    for (int i = 1; i < arr.length; i++) {
      if (arr[i] > max) {
        max = arr[i];
      }
    }
    return max;
  }
}

Multiple Choice Practice

Question 1:

What is the output of the following code?


int[] arr = {1, 2, 3, 4, 5};
for (int i = 0; i < arr.length; i++) {
    arr[i] *= 2;
}
System.out.println(arr[2]);
A. 2
B. 3
C. 6
D. 4
E. 5
Explanation: The answer is C. The loop multiplies each element by 2, so arr[2] (which was 3) becomes 6.

Question 2:

Which of the following is NOT a valid array declaration in Java?

A. int[] arr = new int[5];
B. int[] arr = {1, 2, 3};
C. int arr[] = new int[5];
D. int arr[5];
E. int[] arr = new int[]{1, 2, 3};
Explanation: The answer is D. In Java, you cannot specify the size in the array declaration. The size must be specified when creating the array using the new keyword.

Question 3:

What happens when you try to access an array index that is out of bounds?

A. The program continues with a default value
B. The program returns null
C. An ArrayIndexOutOfBoundsException is thrown
D. The program returns 0
E. The program returns -1
Explanation: The answer is C. When you try to access an array index that is out of bounds, Java throws an ArrayIndexOutOfBoundsException. This is a runtime exception that must be handled using try-catch blocks.

Question 4:

What is the length of the following array?


int[] arr = new int[5];
A. 4
B. 5
C. 6
D. 0
E. Undefined
Explanation: The answer is B. When you create an array with new int[5], it creates an array of length 5. The length is the number of elements the array can hold, and valid indices are 0 to 4.

Question 5:

Which of the following is true about enhanced for loops with arrays?

A. They can modify array elements
B. They can traverse arrays in reverse
C. They can access array indices
D. They can only read array elements
E. They can skip elements
Explanation: The answer is D. Enhanced for loops can only read array elements sequentially. They cannot modify elements, access indices, or traverse in reverse. For these operations, you need to use a traditional for loop.

Tips for Success

  • Remember that array indices start at 0.
  • Always check array bounds before accessing elements.
  • Use array.length to get array size.
  • Practice common array algorithms like searching and sorting.
  • Be careful when modifying array elements in loops.
